# Sprint Process(Workflow Cycle)

---
## Sprint 1:

- The first sprint was a double start for this group. In the beginning we had a vision for a product but realized that there were no data available. We made a plan for another project where we got stuck on choosing what data to use.  

- To get the project moving we narrowed our focus, start small and have the opportunity to build from that. We made a plan for the next sprint and had decided on the data, the product and what to start with.

--- 

## Sprint 2:

- In the second sprint we started with the programing and building of the project. Due to making the tasks to big we got some help on how to narrow them down and get on track on where to begin. 
Tasks became clearer, code started to evolve and the program started to form. A lot of the big parts were built and working but were not able to be connected before the sprint ended.

---

## Sprint 3:

- The third sprint focused on getting a working product. Most components were done from the previous sprint but the connection between them was missing. Finding ways for the data to be ingested into the database upon creation of the environment. Making sure that it is cleaned and filtered without having to do so manually. 

- With these parts in place, it got easier to test the connection to the database which enabled the test of the endpoints. Through localhost we could see the result of the endpoints and the data stored in the database.

--- 

## Sprint 4: 

- In sprint four the focus was now primarily on implementing the correct data from SMHI after we had run all required tests with the test/mock data.

- In their file the first two lines contains information(metadata rows) that are not interesting for this project and also missing column names. 

- The challenge was to ingest the raw data into a table in the database reading the two metadata rows, as well as taking off/droping these two lines.

- Another focus was to implement/visualize “nice to have” diagrams to show the data in a graphic way(graphical representation). For example you can input/enter a year and get a line chart where you see the average temperature for each month. 

---
